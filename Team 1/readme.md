## 02. ARCHITECTURE : 6단계 번역 파이프라인
* **01 원문 입력** : 승정원일기 원문 입력
* **02 NER 추출**  : SillokBERT-NER을 통한 개체명 인식
* **03 KB 대조**   : 지식 베이스(KB) 매핑 및 시기 필터링
* **04 Entity Link**: 팩트 생성 (한자-한글 매핑)
* **05 프롬프트 증강**: NERinject 블록 프롬프트 삽입
* **06 번역 출력** : Gemma-4-26B를 통한 최종 한국어 출력
> **핵심 설계 결정 - 아키텍처 이질성 극복**
> SillokBERT(인코더)와 Gemma(디코더)는 내부 표현 공간 및 레이어 구성이 이질적이라 가중치 직접 이관이 불가능합니다. 이에 따라 지식을 가중치 대신 **프롬프트 형식으로 전달하는 방식**으로 연구 방향을 전면 수정했습니다.

## 04 EXPERIMENT : 4단계 Ablation 실험 설계
> 누적 전략 구조 – 각 개입 효과를 단계적으로 분리 측정 (Gemma-4-26B, temperature = 0.0 설정)
> 비교 모델 군: Qwen3, Phi4-mini, DeepSeek-v3 / 평가셋 300개 층화 샘플링 (KL divergence 기준 길이 ~0.000, 패턴 ~0.001 준수)

### 🔄 단계별 실험 전략 및 파이프라인 구조

1. **Baseline (Zero-shot)**
   * **구조:** 최소 지시문 프롬프트 + 한자 원문만 입력
   * **특징:** '다음 한문을 현대 한국어로 번역하세요.' 수준의 최소 지시문으로 번역문 출력을 유도함
2. **Few-shot Prompting (STYLE)**
   * **구조:** 고정된 5개의 대표(REP) 패턴 예시 문장 스태틱 배치
   * **특징:** 일관성 유지가 다양성보다 중요하여, 동적 선택 방식보다 고정 예시 방식이 더 우수함
3. **Few + NERfix (Post-hoc)**
   * **구조:** 번역 출력 이후 사후 처리(Post-processing) 단계에서 hanja 라이브러리 음독 교정 적용
   * **특징:** 성씨 앵커 기반으로 오독은 탐색·교정할 수 있으나, LLM 번역 과정에서 완전히 누락/생략된 인명(실험 중 34개 케이스)은 사후 복구가 불가능하여 ETS 0.825의 명확한 성능 천장이 존재함
4. **Few + NERinject (Pre-inject / Optimized)**
   * **구조:** 번역 수행 전(Pre-inject) 단계에서 인물 KB를 프롬프트에 직접 주입
   * **특징:** 한자-한글 개체명 추출(SillokBERT-NER) 후 매핑 팩트(e.g., 李景曾: 이경증, 張維: 장유)를 예시 블록과 함께 주입하여 번역 과정에서의 인명 생략 자체를 원천 방지, 최고 수준의 ETS 달성

## 06 METHODOLOGY : Closed Loop 문제 & ETS 설계
> **"평가 지표의 독립성이 결과의 신뢰성을 결정한다"** — 본 연구의 핵심 방법론적 기여

### ❌ 기존 NER Recall 방식의 한계 (닫힌 고리 - Closed Loop)
* 정답지(Ground Truth)를 추출하는 알고리즘과 NERinject에 주입되는 지식 소스가 동일한 사내 인물 KB를 공유함
* 이 경우, 실제 번역문 문맥의 자연스러움이나 언어학적 번역 품질과 완전 무관하게 **'주입되었던 인명이 번역 출력본에 단순히 잔존하는가'**만으로 정답 처리가 됨
* 이는 실제 번역 품질을 반영하지 못한 채 인위적으로 부풀려진 수치인 **NER Recall = 1.000**을 산출하는 구조적 결함을 가짐

###  ETS (Entity Translation Score) - 독립 소스 설계 도입
* 국사편찬위원회 승정원일기 HTML 원본 데이터에서 역사학자들이 직접 수동으로 태깅한 `span.idx_person` 마크업 데이터를 기반으로 정답지를 완벽히 상호 독립적으로 구축
* 모델 및 주입 모듈과 평가 소스를 완전히 분리하여 순환 평가(Closed Loop) 오류를 원천적으로 제거함
* **수식 정의:** $$\text{ETS} = \frac{\text{번역문에 올바르게 등장한 인명 수}}{\text{원문의 총 인명 수}}$$
* 독립 설계 적용 결과, 인위적 수치가 제거된 **실제 객관적 인명 보존율인 0.888**을 정밀하게 측정해 냄

## 07 LIMITATIONS : 연구의 한계 및 향후 과제

1. **KB 커버리지의 도메인 한계**
   * 현재 지식 베이스가 인조·정조 시기(33,002명) 데이터로만 한정되어 있어, 타 왕조의 미등재 인명은 주입 자체가 불가능함 (현재 ETS 0.888이 이론적 한계선인 천장으로 작용)
   * **해향 과제:** 국사편찬위원회 전체 인물 DB 연계 고도화 및 데이터 기반 KB 자동 확장 방법론 연구
2. **평가 지표 대상의 확장 필요성**
   * 현재 개발된 ETS는 오직 '인명'의 보존율 및 번역 정확도만 측정 가능하여 관직명, 지명의 오역 현상은 포착 불가
   * **해향 과제:** 관직명과 지명 개체명까지 종합 반영하는 확장형 Entity Translation Score 지표 개발
3. **단일 LLM 모델 검증의 한계**
   * 본 연구는 Gemma-4-26B 단일 디코더 모델을 중심으로 최적화가 진행되어 타 오픈소스 모델 진영에서의 범용성이 미검증됨
   * **해향 과제:** 승정원일기 전 시기(인조~순종)로 영역 확대 및 다기종 LLM 비교 실험을 통한 범용성 검증

📌 **향후 핵심 로드맵:** SJW 전 시기 평가 확대, KB 미등재 인명에 대한 음독 자동 생성 모델 연구, 반자동 완역 지원 파이프라인 구축

## 08 CONCLUSION : 결론
> 거대한 파인튜닝 비용 없이 오직 **프롬프트 전략과 정확한 KB 주입(NERinject)** 기술만으로도 매우 높은 실용적 번역 성능을 달성했으며, 독립적 평가 방법론(ETS)을 제안하여 연구 신뢰성에 기여함

* **최종 핵심 성과:** `NERinject` 파이프라인 적용 결과 **BLEU(c) 40.61** (+5.10p 대폭 상승), **ETS 0.888** 확보, **KB 등재 정확도 99.4%** 달성
* **3대 핵심 인사이트 요약:**
  1. **일관성 > 다양성:** 번역 도메인 작업 시 Few-shot 예시를 문장마다 동적으로 가변 주입하는 것보다 신중히 정해진 고정 5개 예시(REP 패턴)를 제공하는 것이 모델 맥락 유지에 훨씬 유수함
  2. **문체와 인명 간 트레이드오프 해결:** Few-shot 기법만 쓰면 사료 특유의 문체는 잡히나 인명 유실(ETS 하락)이 발생하는 모순적 현상을, 사전 지식 주입 전략(`NERinject`)을 융합하여 두 마리 토끼를 모두 잡음
  3. **평가 독립성을 통한 신뢰성 확보:** 허수인 NER Recall 1.000 대신 독립 소스 기반의 ETS 0.888 지표를 설계·제안함으로써 고전 문헌 번역 AI 연구의 방법론적 기여를 이뤄냄
* **🚀 기대 파급 효과:**
  * 본 연구의 `NERinject` 파이프라인을 자동화 프로세스에 이식할 경우, 인간 전문 번역가가 검토하기 전 단계인 **'고품질 1차 초안 자동 생성'** 시스템 구축이 가능해짐
  * 결과적으로, 기존 인력 수작업 기준 **2062년으로 판가름 났던 승정원일기 최종 완역 시점을 획기적으로 조기 달성**할 수 있는 도약의 발판을 마련함
